# Финансовая отчётность ООО СПК «Лазурный» — рабочая инструкция

Комплект для ежемесячной управленческой отчётности: выгрузки 1С → нормализация → плоская таблица `FLAT` → дашборды и аналитические записки.

---

## 1. Состав комплекта

| Файл | Что это |
|---|---|
| `normalize_1c.gs` | Скрипт Google Apps Script: раскладывает выгрузки 1С в плоскую таблицу `FLAT` |
| `dashboard_live.html` | «Живой» дашборд: сам читает `FLAT` из Google-таблицы и обновляется |
| `dashboard_svodny_2026.html` | Аналитический дашборд (5 вкладок): глубокий разбор за март–апрель |
| `ОПиУ_март-апрель_2026.docx` | Записка по прибылям и убыткам |
| `Анализ_движения_денег_март-апрель_2026.docx` | Записка по движению денег (ОДДС) |

Два дашборда дополняют друг друга: **живой** — для регулярного мониторинга (обновляется сам), **аналитический** — для разбора (карта стабильности маржи, разнесение ФОТ, воронки; пересобирается при новых данных).

---

## 2. Ежемесячный регламент (главное)

> **Сначала закрыть месяц в 1С → потом выгружать.** Выпуск проведён по факту, отрицательных остатков нет, себестоимость рассчитана. Иначе в отчётность попадут «качели» себестоимости.

Затем:

1. Выгрузить из 1С отчёты: **ОПиУ**, **ОДДС**, **Продажи**, **Валовая прибыль**.
2. Открыть выгрузки как Google-таблицы (для ОПиУ/ОДДС — чтобы сохранилась группировка строк).
3. Скопировать лист с отчётом в таблицу-движок → меню **«🔧 Нормализатор 1С» → «Нормализовать активный лист»**.
4. Повторить для каждого отчёта. Всё копится в листе `FLAT`.
5. Открыть `dashboard_live.html` (с GitHub Pages) → новый месяц появится сам.

Коротко: **закрыл месяц → выгрузил → нормализовал → дашборд обновился.**

---

## 3. Нормализатор 1С (`normalize_1c.gs`)

**Установка (один раз):**
1. Google-таблица → **Расширения → Apps Script**.
2. Вставить код из `normalize_1c.gs`, сохранить.
3. Обновить страницу таблицы → появится меню **«🔧 Нормализатор 1С»**.

**Использование:** встать на лист с выгрузкой → меню → «Нормализовать активный лист». Тип отчёта (ОПиУ / ОДДС / Продажи / Валовая прибыль) определяется автоматически по заголовкам.

**Заголовки 1С** должны быть в первых двух строках (период — в 1-й, показатели — во 2-й) — это стандартный вид выгрузок.

Пункт меню **«Построить сводку (Каналы × Период)»** собирает из `FLAT` лист `СВОДКА` с выручкой/прибылью/рентабельностью.

---

## 4. Таблица `FLAT` (длинный формат)

Одна строка = одно число с метками:

| Отчёт | Период | Канал | Категория | Позиция | Показатель | Значение | Уровень | Загружено |
|---|---|---|---|---|---|---|---|---|

Из этого формата сводной таблицей или формулой `QUERY` собирается любой разрез. Пример — выручка опта по месяцам:
```
=QUERY(FLAT!A:G; "select B, sum(G) where A='ОПиУ' and C='Опт' and F='Доход' group by B")
```

`FLAT` копит историю по месяцам — это и есть ценность. **Не чистить без нужды.** Один месяц грузить **один раз** (повтор добавит дубль — тогда удалить старые строки этого месяца).

## 5. Живой дашборд (`dashboard_live.html`)

Сам читает `FLAT` из вашей опубликованной Google-таблицы. Вкладки: **Обзор · Каналы · Продажи · Деньги**, переключатель месяцев строится автоматически.

Ссылка на ваш `FLAT` уже вписана в файл (строка `SHEET_CSV_URL` в начале скрипта):
```
.../pub?gid=478162687&single=true&output=csv
```
Если опубликуете заново — обновите эту строку (нужна ссылка именно на лист `FLAT`, с `gid`).

### Как открывать — важно

Дашборд тянет данные с Google. Браузер **запрещает** это, если открыть файл **с диска** (`file:///…`) — политика CORS. Поэтому:

- ✅ **Правильно:** выложить `dashboard_live.html` на **GitHub Pages** (там же, где ваш ДДС-дашборд) и открывать по адресу `https://…github.io/…/dashboard_live.html`. Оттуда загрузка с Google работает.
- ❌ **Не сработает:** двойной клик по файлу из «Загрузок» — будет «источник: демо (ошибка загрузки ссылки)».

**Выложить на GitHub Pages:** репозиторий → **Add file → Upload files** → перетащить `dashboard_live.html` → **Commit**. Открыть по адресу Pages.

> Если нужен запуск с диска без GitHub — есть версия загрузки через `gviz/JSONP` (обходит ограничение). Скажите — соберу.

### Что показывает

- **Обзор** — выручка, чистая прибыль, рентабельность за месяц + выручка по каналам.
- **Каналы** — маржа по каналам (выручка − себестоимость) по всем месяцам. Чистая база без «качелей» и без распределения ЗП.
- **Продажи** — топ позиций по выручке за месяц.
- **Деньги** — денежный поток по направлениям (нужен ОДДС с колонками Поступления/Платежи/ЧДП).

## 6. Аналитический дашборд (`dashboard_svodny_2026.html`)

Самодостаточный (данные зашиты внутри), открывается двойным кликом или с GitHub Pages. 5 вкладок: **Март · Апрель · Сравнение · Продажи · Каналы**. Содержит глубокую аналитику: воронки каналов, разнесение ФОТ, мост прибыли и **карту стабильности маржи по моделям** (видно, что «качели» — только в опте, на нерегулярных моделях).

Обновляется не автоматически — пересобирается под новые данные при необходимости.

---

## 7. Дисциплина учёта (чтобы цифры были чистыми)

Из разбора март–апрель: искажения себестоимости — **учётные, а не расчётные**. Финансовый результат считается корректно, но себестоимость отдельных моделей «прыгает» из-за производственного учёта. Чтобы это не попадало в отчётность:

1. Выпуск ГП проводить по факту, **датой не позже отгрузок**.
2. Перед закрытием убрать **отрицательные остатки** склада ГП.
3. Запустить **расчёт себестоимости**.
4. После налаженного выпуска — включить **контроль остатков** (запрет отгрузки в минус).
5. Опционально — **плановая (нормативная) себестоимость** по моделям.

Метод оценки оставить **«средняя»** — менять его не нужно: проблема не в методе, а в своевременности выпуска.

---

## 8. Если что-то не сходится

- **Дашборд показывает «демо (ошибка загрузки)»** → открыт с диска или не включена публикация. См. раздел 5.
- **«опубликован не лист FLAT»** → по ссылке отдаётся другой лист. Использовать ссылку с `gid` именно листа `FLAT` (или сделать `FLAT` первым листом).
- **В `FLAT` каналы пустые / контрагенты не отделились** → прислать скриншот `FLAT`, подстроить парсер (`parseItems` / `parsePnLCash` в скрипте).
- **Цифры в дашборде и записках расходятся** → проверить, что загружена актуальная версия выгрузки и месяц не задвоен в `FLAT`.
